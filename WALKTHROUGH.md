## 🎯 導入の目的

### どんな課題があったか

ECチームが Amazon 用の商品画像を制作した後、**ガイドラインに合っているかを毎回手動で確認**していました。
この作業には以下のような課題がありました。

| 課題 | 具体的な内容 |
|------|-------------|
| ⏰ **時間がかかる** | Amazon Seller Central の公式ガイドライン（面積比85%、純白背景、サイズなど）を1画像ずつ手作業で確認 |
| 👤 **属人化** | 社内の画像品質基準（影、明るさ、質感、構図など）は担当者の経験と感覚に依存しており、判断にブレがある |
| 🔁 **同じ指摘の繰り返し** | Slackでのフィードバックを見ると、同じ修正指摘が何度も発生している |

### 解決策

**「人の目でやっていたチェックを、プログラムで自動化しよう」** というアプローチで、このツールを開発しました。

- 公式ガイドラインの数値基準 → **画像解析で自動判定**
- 社内の品質基準 → **数値指標に変換して自動評価**
- 判断のブレ → **統一された基準で100点満点スコア化**

## 🏗️ システムの概要

### 主要機能 7つ

| # | 機能 | 説明 |
|---|------|------|
| 1 | 📏 **Amazon公式ガイドライン自動チェック（6項目）** | 面積比、白背景、画像サイズ、アスペクト比、影、明るさを自動判定 |
| 2 | 🎨 **社内品質チェック（7項目）** | 質感、立体感、構図、鮮明度、色味、局所暗部、光の方向性を数値で評価 |
| 3 | 💯 **総合スコア（100点満点）** | 全13項目の結果から総合スコアを算出。80点以上で合格 |
| 4 | 📝 **カスタムガイドライン管理** | 社内ガイドラインを誰でも追加・削除できる。GitHub APIでデータを永続保存 |
| 5 | 🏷️ **ブランド別 OK/NG 例集（19ブランド対応）** | GRAV、sopoa、mura など19ブランドごとに良い例・悪い例を蓄積 |
| 6 | 💬 **コメント機能** | 画像ごとにフィードバックコメントを残せる |
| 7 | 📄 **PDFレポート出力** | チェック結果をワンクリックで横型A4のPDFレポートとしてダウンロード |

## ✅ チェック項目の詳細

### Amazon公式ガイドライン（6項目）

| # | チェック項目 | 判定基準 | 判定方法 |
|---|-------------|---------|---------|
| 1 | 商品面積比 | 画像の85%以上を商品が占めること | 商品の外接矩形（バウンディングボックス）の面積比を計算 |
| 2 | 影の有無 | ドロップシャドウや鏡面反射があるか | 商品外の背景上でグレー帯ピクセルを3つの方法で検出 |
| 3 | 画像サイズ | 長辺 500px 以上 10,000px 以下 | ピクセルサイズを直接チェック |
| 4 | 明るさ | 暗すぎず明るすぎないこと | 商品部分の平均輝度（加重平均: R×0.299 + G×0.587 + B×0.114）を計算 |
| 5 | 白背景 | 背景が純白（RGB: 255,255,255）であること | 四隅＋辺の中央の8箇所をサンプリングして判定 |
| 6 | アスペクト比 | 正方形〜やや縦長が推奨 | 幅÷高さの比率をチェック（0.6〜1.05が理想） |

### 社内品質チェック（7項目）

| # | チェック項目 | 何を見ているか | 数値指標 |
|---|-------------|---------------|---------|
| 7 | 質感 | 素材感が伝わるかどうか | 商品部分の輝度の標準偏差＋局所コントラスト（5x5ブロック） |
| 8 | 立体感 | 平面的に見えないか | 上位5%と下位5%の輝度差（ダイナミックレンジ） |
| 9 | 構図 | 傾いていないか、中央にあるか | 慣性モーメントから主軸角度を計算＋重心のオフセット |
| 10 | 鮮明度 | ぼやけていないか | ラプラシアンフィルタ（二次微分）の分散値 |
| 11 | 色味 | くすんでいないか、色温度は適切か | RGB間の差（彩度）＋ R-B差（色温度） |
| 12 | 局所暗部 | 商品の一部だけ暗くないか | 6x6グリッドに分割して各ブロックの輝度を比較 |
| 13 | 光の方向性 | 自然な陰影があるか | 商品を4象限に分割して輝度の差を比較（左上が明るいのが理想） |

### 採点ルール

```
OK項目  → 1.0点
注意項目 → 0.2点（厳しめ評価）
NG項目  → 0点 + NG1件につき追加で -5点のペナルティ

総合スコア = (合計ポイント ÷ 全項目数) × 100 − ペナルティ
合格ライン = 80点以上
```

## 🛠️ 使用テクノロジー

### ファイル構成

```
amazon-image-checker/
├── app.py                 … Streamlit WebアプリのUI・メイン画面
├── image_checker.py       … 13項目のチェックロジック（画像解析の心臓部）
├── pdf_report.py          … PDFレポート生成（横型A4・日本語対応）
├── requirements.txt       … 依存パッケージ一覧
├── start.command          … ローカル起動用スクリプト（macOS）
├── .streamlit/
│   └── secrets.toml       … GitHub APIトークン（※ Git管理外）
└── .gitignore             … secrets.toml等を除外
```

### 技術スタック

| 技術 | 役割 | 選定理由 |
|------|------|---------|
| **Streamlit** | WebアプリのUI | Pythonだけでインタラクティブなアプリが作れる。デプロイも無料 |
| **Pillow** | 画像の読み込み・描画 | 軽量な画像処理ライブラリ。赤枠描画などに使用 |
| **NumPy** | 画像解析・数値計算 | ピクセルデータを配列として高速に処理。OpenCVなしで実現 |
| **ReportLab** | PDF生成 | 日本語CIDフォント（HeiseiKakuGo-W5）対応。横型A4レポート |
| **GitHub API** | カスタムデータの永続化 | ガイドライン・OK/NG例集・コメントをGitHubのdataブランチに保存 |
| **Streamlit Community Cloud** | ホスティング | 無料でWebアプリを公開。GitHubリポジトリと自動連携 |

### ポイント: OpenCVを使わない設計

画像解析には一般的に OpenCV（cv2）を使うことが多いですが、このツールでは **Pillow + NumPy だけで全チェックを実装**しています。

- ラプラシアンフィルタ（鮮明度チェック）→ NumPyの配列演算で手動実装
- エッジ領域の検出（膨張・収縮）→ NumPyのroll関数で手動実装
- 商品マスクの生成（背景除去）→ 閾値ベースのピクセル分類

これにより **依存パッケージが最小限** になり、Streamlit Cloud へのデプロイもスムーズです。

## 🔄 動作フロー

```
┌─────────────────────────────────────────────────────────────────┐
│                    Amazon商品画像チェッカー                       │
└─────────────────────────────────────────────────────────────────┘

  ① 画像アップロード（複数枚OK / PNG, JPG, JPEG, WebP対応）
          │
          ▼
  ② 商品領域の検出
     ├─ 透過PNG → アルファチャンネルで商品を特定
     └─ 白背景画像 → RGB閾値（240以上）で背景を除去
          │
          ▼
  ③ 13項目の自動チェック実行
     ┌────────────────────────────┐
     │ Amazon公式（6項目）          │
     │  面積比 / 影 / サイズ /      │
     │  明るさ / 白背景 / 比率      │
     ├────────────────────────────┤
     │ 社内品質（7項目）            │
     │  質感 / 立体感 / 構図 /      │
     │  鮮明度 / 色味 / 局所暗部 /  │
     │  光の方向性                  │
     └────────────────────────────┘
          │
          ▼
  ④ 結果表示
     ├─ 総合スコア（100点満点 / 80点で合格）
     ├─ 商品に赤枠を描画した画像プレビュー
     └─ 各項目ごとの OK / 注意 / NG 表示
          │
          ▼
  ⑤ コメント追加（任意）
     └─ 画像ごとにフィードバックを記録
          │
          ▼
  ⑥ PDFレポートをワンクリックでダウンロード
     └─ 横型A4 / 日本語対応 / サマリー＋詳細
```

### データの永続化フロー

```
カスタムデータ（ガイドライン / OK・NG例 / コメント）
          │
          ├─→ ローカルJSON保存（常に）
          │
          └─→ GitHub API 経由で data ブランチに保存
              （secrets.toml に GitHub トークンがある場合）
```

## 📝 補足情報

### セキュリティ

- `.streamlit/secrets.toml`（GitHub APIトークン）は `.gitignore` で除外済み

### ローカルでの起動方法

```bash
# macOS の場合
# start.command をダブルクリック、または：
cd /path/to/amazon-image-checker
pip3 install -r requirements.txt
python3 -m streamlit run app.py --server.port 8502
```
