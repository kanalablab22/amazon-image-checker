# 🔍 Amazon商品画像チェッカー

商品画像をアップロードするだけで、**Amazonガイドライン＋社内基準** に基づいて自動チェックしてくれるWebアプリです。

> 🌐 **Webアプリなので、ブラウザがあればすぐ使えます（インストール不要！）**
>
> 👉 **https://amazon-image-checker-s8g6qqtknbdzfqhkcn884t.streamlit.app/**

---

## ✨ 主な機能

| # | 機能 | 説明 |
|---|------|------|
| 1 | 📏 **Amazon公式ガイドラインチェック** | 面積比85%、白背景、影、画像サイズ（500〜10,000px）、明るさ、アスペクト比を自動判定 |
| 2 | 🏢 **社内品質チェック** | 質感、立体感、構図/傾き、エッジ鮮明度、色味、局所暗部、光の方向性をチェック |
| 3 | 💯 **総合スコア** | 100点満点で採点（80点以上が合格ライン） |
| 4 | 📝 **社内ガイドラインの追加・削除** | チーム独自のチェック項目を自由に追加OK。GitHubに永続保存されるので消えない |
| 5 | 🏷️ **ブランド別 OK例集・NG例集** | ブランドごとに「こういう画像はOK」「こういうのはNG」を蓄積できる |
| 6 | 💬 **画像ごとのコメント機能** | 「影をもう少し強くしたほうが良さそう」など、画像単位でメモを残せる |
| 7 | 📄 **PDFレポート出力** | 横型A4・日本語対応のレポートをワンクリックでダウンロード |

## 🚀 使い方

### ステップ1: アプリを開く

ブラウザで以下のURLにアクセスしてください。

👉 **https://amazon-image-checker-s8g6qqtknbdzfqhkcn884t.streamlit.app/**

### ステップ2: 画像をアップロード

1. 画面中央の **アップロードエリア** に画像をドラッグ＆ドロップ（複数枚OK！）
2. 自動でチェックが始まります

### ステップ3: 結果を確認

- 📊 **総合スコア**（100点満点）が表示されます
- ✅ 合格 / ⚠️ 注意 / ❌ NG の3段階で各項目を判定
- 商品領域が **赤枠** で表示されるので、面積比が直感的にわかります

### ステップ4: PDFレポートをダウンロード（任意）

画面下部の 📥 **PDFレポートをダウンロード** ボタンを押すと、横型A4のレポートが生成されます。

## 📋 サイドバーの使い方

画面左側のサイドバーには、以下の機能があります。

### 📋 Amazon公式ガイドライン

Amazon Seller Centralの公式ガイドラインをチェックリストとして表示。手動で確認しながらチェックを入れられます。

### 🏢 社内ガイドライン

- あらかじめ登録されている社内チェック項目が表示されます
- **➕ 追加** ボタンで、チーム独自のチェック項目を追加できます
- 追加した項目はGitHubに保存されるので、誰がアクセスしても同じ内容が見えます

### 🏷️ ブランド別 OK / NG例集

1. プルダウンからブランドを選択
2. そのブランドのOK例・NG例が表示されます
3. 新しいブランドの追加もできます
4. 「✅ OK例」「❌ NG例」を自由に登録・削除できます

## 📊 チェック項目の詳細

### Amazon公式ガイドライン系（6項目）

| チェック項目 | 基準 |
|-------------|------|
| 商品面積比 | 画像の85%以上を商品が占めているか |
| 白背景 | 背景がRGB(255,255,255)の純白に近いか |
| 影の検出 | 不自然な影がないか |
| 画像サイズ | 長辺が500px以上〜10,000px以下か |
| 明るさ | 画像全体が適切な明るさか |
| アスペクト比 | 縦横比が適切か |

### 社内品質チェック系（7項目）

| チェック項目 | 内容 |
|-------------|------|
| 質感 | 商品の素材感が伝わるか |
| 立体感 | 平面的でなく立体的に見えるか |
| 構図/傾き | 商品が傾いていないか、構図は適切か |
| エッジ鮮明度 | 商品の輪郭がくっきりしているか |
| 色味 | 色が自然で鮮やかか |
| 局所暗部 | 商品の一部だけ暗くなっていないか |
| 光の方向性 | 光の当たり方が自然か |

## 🔧 ローカルで開発する場合

> 通常の利用ではこの手順は不要です。開発・カスタマイズしたい場合のみお読みください。

### 必要な環境

| 項目 | バージョン |
|------|-----------|
| Python | 3.14 以上 |
| pip | 最新版推奨 |

### セットアップ（macOS）

```bash
# 1. リポジトリをクローン（コードをダウンロード）
git clone https://github.com/kanalablab22/amazon-image-checker.git
cd amazon-image-checker

# 2. 依存パッケージをインストール
pip3 install -r requirements.txt

# 3. アプリを起動
python3 -m streamlit run app.py
```

または、`start.command` をダブルクリックするだけでもOKです！（依存パッケージのインストールも自動でやってくれます）

### セットアップ（Windows）

```powershell
# 1. リポジトリをクローン（コードをダウンロード）
git clone https://github.com/kanalablab22/amazon-image-checker.git
cd amazon-image-checker

# 2. 依存パッケージをインストール
pip install -r requirements.txt

# 3. アプリを起動
python -m streamlit run app.py
```

起動すると、ブラウザで `http://localhost:8502` が自動的に開きます。

### データの永続化について

ローカル実行時は、社内ガイドラインやOK/NG例集がローカルのJSONファイルに保存されます。

本番環境（Streamlit Cloud）では、**GitHub API** を使って `data` ブランチにJSONファイルとして保存されるため、全員で共有できます。

Streamlit Cloud の **Secrets** に以下を設定してください：

```toml
[github]
token = "ghp_xxxxx"  # GitHubのPersonal Access Token
repo = "kanalablab22/amazon-image-checker"
```

## ❓ トラブルシューティング

### Q: アプリが開けない / 画面が真っ白になる

**A:** ブラウザのキャッシュをクリアして、もう一度アクセスしてみてください。それでもダメな場合は、別のブラウザ（Chrome推奨）で試してみてください。

### Q: 画像をアップロードしてもチェックが始まらない

**A:** 対応形式は **JPEG / PNG / WebP** です。それ以外の形式（BMP、GIFなど）は対応していません。

### Q: 社内ガイドラインを追加したのに消えてしまった

**A:** 本番環境ではGitHubに自動保存されるので消えません。ローカル実行の場合はローカルのJSONファイルに保存されますが、ファイルを削除すると消えてしまいます。

### Q: PDFの日本語が文字化けする

**A:** アプリ側で日本語フォント（HeiseiKakuGo-W5）を使用しているので、通常は文字化けしません。もし文字化けする場合は、PDFビューアを変更してみてください（Adobe Acrobat Reader推奨）。

### Q: ローカルで起動したら「ModuleNotFoundError」と出る

**A:** 依存パッケージがインストールされていない可能性があります。以下を実行してください：

```bash
# macOS
pip3 install -r requirements.txt

# Windows
pip install -r requirements.txt
```

## 💡 よくある質問

### Q: スマートフォンからも使えますか？

**A:** はい！ブラウザでアクセスすれば使えます。ただし、画面が小さいのでPC推奨です。

### Q: チェック結果の「80点以上で合格」は変更できますか？

**A:** 現在は80点固定です。変更したい場合は `app.py` のスコア表示部分を編集してください。

### Q: 一度にアップロードできる枚数に上限はありますか？

**A:** Streamlitの仕様上、1ファイルあたり200MBが上限です。枚数に明確な上限はありませんが、10枚程度までが快適に動作します。

### Q: 誰かが追加したガイドラインを元に戻せますか？

**A:** GitHubの `data` ブランチに履歴が残っているので、GitHub上から過去の状態に戻すことができます。
